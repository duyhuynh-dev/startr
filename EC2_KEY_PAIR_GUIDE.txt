WHAT IS A KEY PAIR?
===================
A key pair is like a password, but more secure. It has two parts:
- Public key: Stored on AWS (allows you to connect)
- Private key: You download (.pem file) - KEEP IT SECRET!

It's used to securely SSH into your EC2 instance without a password.

AFTER CREATING - IMPORTANT STEPS:
==================================

Step 1: Save the downloaded .pem file
--------------------------------------
- It downloads to your Downloads folder
- Move it to ~/.ssh/ folder (recommended):

  mv ~/Downloads/vc-matcher-backend.pem ~/.ssh/

Step 2: Set secure permissions
-------------------------------
The key file MUST have restrictive permissions:

  chmod 400 ~/.ssh/vc-matcher-backend.pem

This makes it readable only by you (required by SSH).

Step 3: Use it to connect
--------------------------
After your EC2 instance is running:

  ssh -i ~/.ssh/vc-matcher-backend.pem ubuntu@YOUR_EC2_IP

Replace YOUR_EC2_IP with your actual EC2 public IP address.


TROUBLESHOOTING:
================

"Permission denied (publickey)" error?
- Make sure permissions are 400: chmod 400 ~/.ssh/your-key.pem
- Verify you're using the correct key name
- Check the key pair is selected in EC2 instance settings

"WARNING: UNPROTECTED PRIVATE KEY FILE!"
- Run: chmod 400 ~/.ssh/your-key.pem

Can't find the downloaded file?
- Check your Downloads folder
- Check browser download history
- If lost, create a new key pair (but won't work with existing instance)
